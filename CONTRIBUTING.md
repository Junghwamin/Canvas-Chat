# 기여 및 개발 가이드라인 (CONTRIBUTING)

이 프로젝트는 `CLAUDE.md`를 중심으로 개발 이력과 변경 사항을 철저히 관리합니다. 모든 개발 작업(AI Agent 포함)은 다음 규칙을 준수해야 합니다.

---

## 📜 CLAUDE.md 관리 규칙

**모든 주요 변경 사항, 버그 수정, 에러 발생 및 해결 과정은 반드시 `CLAUDE.md`에 기록해야 합니다.**

### 1. 기록 시점
- 새로운 기능을 구현했을 때
- 기존 코드를 수정하거나 리팩토링했을 때
- 버그를 수정했을 때
- 개발 중 발생한 주요 에러와 해결 방법을 찾았을 때
- 환경 설정이나 패키지가 변경되었을 때

### 2. 기록 형식

각 업데이트는 날짜별로 구분하며, 다음 카테고리를 사용하여 체계적으로 정리합니다.

```markdown
## [YYYY-MM-DD] 업데이트 제목

### 🚀 기능 추가 (Feat)
- 기능 내용 설명
- 관련 파일: `path/to/file`

### 🛠 수정 사항 (Fix/Refactor)
- 수정 내용 설명
- 관련 파일: `path/to/file`

### 🐛 버그 및 에러 해결 (Troubleshooting)
- **증상**: 에러 메시지 또는 현상
- **원인**: 파악된 원인
- **해결**: 적용한 해결 방법
- 관련 파일: `path/to/file`

### 📦 패키지 및 환경 (Chore)
- 추가된 패키지 또는 환경 변수 변경 사항
```

### 3. 작업 전 필독 사항
개발을 시작하기 전에 반드시 `CLAUDE.md`의 최근 기록을 확인하여 프로젝트의 현재 상태와 이전 이슈를 파악해야 합니다.

---

## 🔄 버전 관리 전략

1. **Git Commit**: `CLAUDE.md` 업데이트는 해당 코드 변경 사항과 함께 커밋합니다.
2. **Release**: 주요 버전 업데이트 시 `CLAUDE.md`의 내용을 바탕으로 `CHANGELOG.md` 또는 릴리스 노트를 작성합니다.

---

## ⚠️ AI Agent 지침

이 프로젝트에 참여하는 모든 AI Agent는 다음을 수행해야 합니다:
1. 작업을 완료하기 전, `CLAUDE.md` 파일을 읽고 현재 기록 양식을 확인한다.
2. 작업 내용을 요약하여 `CLAUDE.md` 최상단(또는 해당 날짜 섹션)에 추가한다.
3. 특히 **발생한 에러와 해결 과정**을 상세히 기록하여 추후 동일한 문제 발생 시 참고할 수 있도록 한다.
